## 评审背景

PalmTag 是一个“掌纹人格标签”娱乐产品，不做传统手相命运、婚姻、健康、财运、寿命判断。当前技术路线是：

```text
真实手掌图片
-> qwen3.6-plus 视觉识别 33 个掌纹字段
-> 三次识别投票融合
-> 8 母型评分
-> 36 人格匹配
-> 易混人格分流
-> persona_id 调用展示文案
```

当前已完成 V0.1 到 V0.10，最新 9 张真实图结果已按人工判断校准到目标输出。

## 请从以下角度评审

### 1. 产品逻辑

- PalmTag 当前定位是否清晰？
- “掌纹人格标签”是否和传统手相区分开？
- 36 型人格是否适合作为娱乐人格产品？
- 当前结果是否有分享传播潜力？

### 2. 数据结构

- 33 字段是否合理？
- 三层结构是否清晰？
- 字段层、规则层、展示层是否解耦？
- 是否存在字段冗余或缺失？

### 3. 规则引擎

- 8 母型评分是否合理？
- 36 人格匹配逻辑是否合理？
- P20 / P25 / P31 / P32 边界是否合理？
- P31 / P25 是否仍有默认出口风险？
- M8 / P32 是否仍有误触发风险？

### 4. 视觉识别

- 当前选择 `qwen3.6-plus` 是否合理？
- Prompt 校准方向是否正确？
- 视觉模型输出 33 字段是否可行？
- 哪些字段不适合让 VLM 强判？
- 哪些字段应该降权或转为辅助？

### 5. 真实样本测试

- 9 张图样本量是否足够当前阶段判断？
- 当前人工评审方法是否合理？
- 下一步应该扩大样本还是继续修规则？
- 是否需要引入用户主观反馈？

### 6. 上线风险

- 哪些问题会影响结果准确感？
- 哪些问题会影响用户信任？
- 哪些问题会影响裂变传播？
- 是否现在可以进入 UI / H5 开发？
- 如果不能，还需要补哪一步？

## 请输出

请按以下结构输出评审结果：

1. 总体评分：1-10 分。
2. 当前阶段是否可以进入前端 MVP。
3. 最大的 5 个问题。
4. 必须马上修的 3 个问题。
5. 可以以后修的问题。

10. 下一步最合理路线。

请尽量具体，不要只给泛泛建议。尤其请重点审查：

- P20 / P25 / P31 / P32 边界是否过拟合 9 张样本。
- 33 字段中哪些字段对 VLM 来说不稳定。
- 当前是否可以进入 UI/H5，还是应该先扩大样本。
