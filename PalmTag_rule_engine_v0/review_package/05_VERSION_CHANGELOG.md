# 版本演进汇总

## V0.1：规则引擎最小原型

- 目的：验证 Excel -> JSON -> 规则引擎 -> 展示文案链路。
- 内容：建立项目结构，生成 33 字段、母型规则、人格规则、易混分流、展示文案 JSON，实现规则引擎。
- 测试：pytest 通过，10 组样本可跑。
- 问题：只验证技术闭环，未验证规则分布。
- 结论：技术原型成立。

## V0.2：随机 JSON 分布修正

- 目的：检查规则分布是否健康。
- 内容：随机生成 1000 组完整 33 字段输入，修 P25/P29 0 命中、M8 偏高、M7 偏低、HEART_LINE_DEPTH 影响过高。
- 测试：1000 组跑通，P01-P36 均可命中。
- 问题：部分低命中人格仍需后续观察。
- 结论：规则分布进入可用状态。

## V0.3：半自动真实图片测试

- 目的：准备真实手掌图片测试。
- 内容：图片目录检查、视觉 Prompt 草案、三次识别投票、半自动任务文件。
- 测试：若无视觉能力则生成手动识别任务。
- 问题：尚未接真实视觉 API。
- 结论：真实图片测试准备完成。

## V0.4：接入千问 qwen-vl-plus

- 目的：打通图片 -> 视觉模型 -> 33 字段 -> 规则引擎链路。
- 内容：接入阿里云百炼 OpenAI 兼容接口，默认 qwen-vl-plus。
- 测试：链路跑通。
- 问题：9 张真实图全部 M1，P06/P31 集中。
- 结论：技术链路通过，视觉识别结果不通过。

## V0.5：视觉 Prompt 初次校准

- 目的：诊断 qwen-vl-plus 为什么集中输出 M1。
- 内容：统计 33 字段分布，修 OVERALL_CLARITY、HEAD_LINE_DEPTH、LINE_COMPLEXITY 等 Prompt 尺度。
- 测试：重新跑真实图片并对比。
- 问题：FATE_LINE_CLARITY 被压得过低，OVERALL_CLARITY 仍偏高。
- 结论：Prompt 需要人工基准对齐。

## V0.6：人工基准对齐

- 目的：用 9 张人工基准标注校准视觉字段。
- 内容：新增人工标注文件，对比 V0.5 与人工均值，修 Prompt 和后处理策略。
- 测试：重新跑 9 张图。
- 问题：模型选择仍不确定。
- 结论：需要多模型对比。

## V0.7：多模型对比

- 目的：比较 qwen-vl-plus、qwen3-vl-plus、qwen3.5、qwen3.6 系列模型。
- 内容：批量模型对比脚本，生成字段误差和人格分布报告。
- 测试：至少 6 个模型对比。
- 问题：部分模型字段完整率或分布偏差不理想。
- 结论：推荐 qwen3.6-plus。

## V0.8：锁定 qwen3.6-plus 生成评审包

- 目的：基于 qwen3.6-plus 生成 9 张真实图人工评审包。
- 内容：默认模型改为 qwen3.6-plus，整理 review pack。
- 测试：pytest 通过，9 张图评审包生成。
- 问题：需要人工判断结果命中感。
- 结论：进入人工评审阶段。

## V0.9：Prompt + P31/P32 边界修正

- 目的：修 P31 默认出口、P32/M8 误触发、HEAD_LIFE_GAP 长期为 0 等问题。
- 内容：新增 Prompt v09，小幅调整 P31/P25/P32/M8 边界。
- 测试：9 张图跑通，API 失败 0，JSON 失败 0，pytest 50 passed。
- 问题：dayi-left 和 grand-right 仍需人工边界修正。
- 结论：大部分目标达成，但两例需 V0.10 修正。

## V0.10：P20/P25/P31 极小边界修正

- 目的：根据人工判断修两个误判。
- 内容：dayi-left 修回 P31；grand-right 修回 P20；保护 P29/P32。
- 测试：pytest 51 passed；9/9 达成人工目标；27 个 pass 全部 33/33。
- 问题：样本量仍小。
- 结论：当前阶段可进入外部评审。
