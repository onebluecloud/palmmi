# PalmTag Stage 2 UI Guideline

## 设计来源

本指南结合 `huashu-design` 和 `awesome-design-md` 输出，用于 Stage 2 的基础 UI 设计和后续页面实现。

`huashu-design` 对本项目的约束是：从已有产品定位和视觉上下文出发，用掌纹线条、清晰层级和诚实 placeholder 建立质感，避免通用渐变、装饰性堆料和无依据的内容填充。

`awesome-design-md` 对本项目的约束是：把视觉主题、颜色角色、字体层级、组件规则、布局原则、响应式行为和禁区写成可执行的设计规范，而不是只写抽象形容词。

```text
高级感

```

## 页面风格

```text
移动端优先
卡片式布局
大标题强钩子
短文案
高对比 CTA
结果页适合截图
海报页适合分享
```

## 视觉基调

整体风格应神秘但不迷信。可以使用深色底、柔和纸色、玉绿色或冷调高光作为掌纹线条与 CTA 的视觉信号，但避免恐怖、血色、符咒、老派算命和廉价抽奖感。

掌纹视觉优先使用线条、拓扑、扫描轨迹或局部纹理，不要大面积使用真实手掌图。上传页可以出现拍摄示意，但应以线框和说明为主。

结果页要有“被说中”的阅读感，优先突出人格名称、金句和最终判断。解释内容要短、准、可截图。

海报页要像社交资产，不像网页截图。竖版卡片、人格名、金句、掌纹线条、二维码占位和 PalmTag 品牌露出是必备元素。

## 组件和布局约束

移动端优先，默认按手机浏览设计，再向桌面扩展。

主要信息使用卡片式布局，但不要把页面做成一堆装饰卡片。卡片用于结果、上传预览、海报预览、隐私条款分组和打赏二维码承载。

CTA 必须高对比、文案明确。首页主按钮指向上传页，结果页按钮包含生成海报、重新测试、打赏。

文案必须短。页面上不放复杂说明书，不把隐私、技术限制、人格数据细节塞进主流程。

缺失内容使用 `TODO：需要人工补充`，不要用虚构文案填充。

## 主要页面设计要求

### 首页

```text
一句话讲清楚产品
突出“上传掌纹，生成你的人格标签”
主按钮明确
有轻量掌纹线条视觉
```

首页第一屏应让用户立即理解 PalmTag 是轻娱乐人格标签生成器。不要做传统玄学首页，也不要做营销长页。

### 上传页

```text
拍摄说明清楚
上传区域明显
图片预览清晰
错误提示友好
```

上传页只做本地图片选择和预览。拍摄说明要告诉用户保持掌心完整、纹理清晰、光线自然。错误提示不用强烈红色恐吓，保持友好、可重试。

### 分析中页面

```text
有加载状态
有掌纹扫描感
不要伪装成医学检测
```

分析中页面可以用掌纹线条、扫描光、进度文案和短暂停留营造过程感。文案必须明确是娱乐分析，不使用医学、诊断、检测、风险评估等严肃词。

### 结果页

```text
人格名称突出
金句突出
最终判断清晰
掌纹特征作为解释依据
按钮包含：生成海报、重新测试、打赏
```

结果页第一眼回答“我是什么人格”。人格名称和金句是最强层级，解释依据放在次级区域。人格内容只从读取层获得，缺失字段显示 `TODO：需要人工补充`。

### 海报页

```text
竖版卡片
人格名称强突出
金句醒目
掌纹线条装饰
二维码占位
PalmTag 品牌露出
```

海报页只做预览，不做正式 PNG 导出。卡片比例优先适合手机截图和微信传播，避免信息过满。

### 打赏页

```text
轻量
不强迫
二维码占位
文案克制
```

打赏页不制造压力，不使用付费墙语气。文案表达“喜欢可以支持”，二维码为占位，不接真实支付系统。

### 隐私页

```text
说明娱乐用途
说明图片不长期保存
说明最多缓存 24 小时
说明不用于严肃判断
```

隐私页必须说明 PalmTag 是娱乐用途，不用于医疗、心理、身份识别或严肃判断。Stage 2 不做真实上传，后续如出现缓存机制，最多缓存 24 小时。

## 禁止方向

不要做成医院检测风。

不要做成老派算命风。

不要做成廉价 H5 抽奖风。

不要使用恐怖、诡异、血色、符咒感太强的视觉。

不要大面积使用真实手掌图。

不要为了炫技破坏首页、上传、分析、结果、海报、打赏、隐私的基础流程。
