# PalmTag Stage 2 Context

## 项目定位

PalmTag 是一个掌纹人格标签生成器，也是一个轻量娱乐化的人格分析工具。产品形态接近 MBTI 式人格标签，但输入方式来自掌纹图片，输出重点是结果页阅读体验和人格海报传播。

PalmTag 的传播场景优先考虑微信和朋友圈。早期版本免费使用，只保留克制的打赏入口，不做会员、付费解锁或复杂商业化流程。

PalmTag 不是严肃手相算命、医疗诊断、心理诊断、传统玄学算命站、复杂社交产品或付费会员产品。

## Stage 1 上下文

现有 Stage 1 资料以规则引擎验证为主，核心链路是：手动输入掌纹字段，计算母型，匹配 36 人格，处理易混人格分流，并读取展示文案。

Stage 1 明确不包含图片识别、VLM、网页 UI、上传页面或海报生成。Stage 2 必须基于这个方向继续做页面骨架和 mock 流程，不能改变产品定位。

当前已有视觉探索文件 `palmtag-visual-direction.html`，其方向是高级、克制、移动端友好、掌纹线条化、结果卡片和海报可分享。Stage 2 的 UI 基础设计应延续这个方向，但不要把该文件当成生产页面直接改造。

## Stage 2 目标

Stage 2 的目标是完成 MVP 页面骨架、基础 UI 设计和页面流转，让用户可以从首页进入上传页，完成本地预览，通过 mock 分析进入结果页，再进入海报预览页、打赏页和隐私说明页。

本阶段优先验证产品流程、信息架构、移动端阅读体验和基础视觉调性。它不是完整上线版，也不要求真实识别、真实上传、真实支付或复杂后台。

## 本阶段边界

本阶段只做页面骨架和 mock 流程。

本阶段不接真实千问 API，不做真实图片上传服务器，不做登录系统，不做支付系统，不做复杂后台。

本阶段不得修改 `data/personas/36-types.json` 中已有的人格内容，不得自动补充缺失的人格字段，不得硬编码任何人格文案。

如果人格内容缺失，页面统一显示：

```text
TODO：需要人工补充
```

## 设计 Skill 使用

Stage 2 使用 `huashu-design` 和 `awesome-design-md` 进行 UI 基础设计约束。

`huashu-design` 用于约束视觉产出要从已有上下文出发，避免通用 AI 设计感，强调诚实 placeholder、清晰视觉层级、掌纹线条作为真实视觉锚点，而不是堆叠装饰。

`awesome-design-md` 用于吸收 DESIGN.md 式的可读设计规范结构，把视觉主题、颜色角色、字体层级、组件样式、布局原则、响应式行为和禁区写成后续页面实现可读取的文档。
