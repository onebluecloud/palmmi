# Palmmi Stage 3E-b Prompt 验收与冻结标准

## 1. 目标

本文定义生产 Prompt 冻结标准。

Prompt 只有经过实测并达到本文件标准后，才能从生产候选版冻结为生产版。

## 2. 20 张图最低标准

连续 20 张图实测时，生产 Prompt 冻结最低标准为：

| 指标 | 最低标准 |
|---|---:|
| JSON 可解析率 | ≥ 90% |
| Schema 合规率 | ≥ 85% |
| 字段缺失率 | ≤ 10% |
| 枚举越界率 | ≤ 5% |
| 类型错误率 | ≤ 5% |

额外硬性要求：

- 非手掌 / 低质量图不能强行输出高置信掌纹结论。
- VLM 不输出人格。
- VLM 不输出母型。
- VLM 不输出命运判断。
- VLM 不输出健康、婚姻、寿命判断。

## 3. 10 张图暂定标准

如果当前阶段只有 10 张图，暂定标准为：

| 指标 | 暂定标准 |
|---|---:|
| JSON 可解析率 | ≥ 90% |
| Schema 合规率 | ≥ 80% |
| 完全非 JSON 输出 | 不允许连续出现 2 次 |

10 张图只适合做候选版筛选，不建议直接冻结最终生产版。

## 4. 结论分类

| 结论 | 定义 | 后续动作 |
|---|---|---|
| 通过 | 达到 20 张图最低标准，且无硬性违规输出 | 可冻结为 `prompt_v1_final` |
| 有条件通过 | 达到主要指标，但存在少量低质量样本不稳定或 10 张图样本量不足 | 进入小幅 Prompt 修改或补测 |
| 不通过 | JSON、Schema、字段缺失、枚举、类型指标未达标，或出现人格 / 母型 / 命运类输出 | 修改 Prompt 后重新实测 |

## 5. 失败后的处理

失败后按问题类型处理：

| 失败类型 | 处理方式 |
|---|---|
| Markdown 包裹或解释文字 | 强化 JSON 约束 |
| 缺失 key | 明确所有 key 必须出现 |
| 字符串数字或中文枚举 | 强化数值类型约束 |
| 枚举越界 | 缩小字段说明，明确每类字段允许值 |
| 非手掌图被强行解释 | 增加反例约束 |
| 低质量图高置信输出 | 增加低质量保护 |
| 低置信字段不稳定 | 必要时降低部分低置信字段要求 |

可选修改方向：

- 修改 Prompt 措辞。
- 强化 JSON 约束。
- 缩小字段说明。
- 增加反例约束。
- 必要时降低部分低置信字段要求。

## 6. 冻结要求

冻结生产 Prompt 前必须记录：

- 测试时间。
- 模型名称。
- `prompt_version`。
- 样本数量。
- 各项指标。
- 失败样本清单。
- 是否满足冻结标准。
- 冻结后的 Prompt 文件路径。

Prompt 冻结后，后续修改必须走变更记录，不得静默改动。
