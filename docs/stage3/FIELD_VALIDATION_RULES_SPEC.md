# Palmmi Stage 3E 字段校验规则规范

## 1. 目标

本文定义 Palmmi Stage 3E 的字段校验规则，用于把 VLM 原始输出和几何字段输出转换为可审计、可降级、可进入后续匹配前置流程的 33 字段对象。

3E 只定义校验策略，不实现校验代码，不接 API，不做规则引擎，不做人格匹配。

## 2. 校验输入与输出

输入来源：

- VLM 原始输出 JSON。
- MediaPipe / 几何处理输出字段。

输出目标：

- 只包含 33 个大写字段的 Palmmi 内部标准化对象。
- 每个字段 key 必须存在。
- 每个字段值必须是合法数字枚举或 `null`。
- 校验与降级记录必须可追踪。

## 3. 校验层级

### 3.1 JSON 可解析性校验

VLM 原始输出必须先通过 JSON 解析。

- 如果输出带 Markdown 包裹，应先进入提取/重试分支，不得直接进入字段映射。
- 如果输出带解释文字，应先进入提取/重试分支，不得直接进入字段映射。
- 如果完全不是 JSON，返回 `JSON_PARSE_FAILED`。

### 3.2 必填 key 存在性校验

Palmmi 内部标准化层的 33 个大写字段全部是必填 key。

- key 缺失比 key 存在但 value 为 `null` 更严重。
- 缺失字段不能直接流入规则引擎。
- 缺失字段必须进入重试、降级或重拍分支。

### 3.3 字段名白名单校验

Palmmi 内部标准化层只允许 33 个大写字段名。

- 未定义字段名不得进入标准化层。
- V4.2 字段表外字段不得进入标准化层。
- VLM 小写字段只允许在原始输出层存在，不能直接作为内部字段。

### 3.4 字段类型校验

33 个内部字段只允许 `number` 或 `null`。

- 字符串数字不允许，例如 `"2"`。
- 中文枚举不允许，例如 `"清晰"`。
- 布尔值不允许，例如 `true` / `false`。
- 数组、对象不允许。
- 类型错误必须视为校验失败。

### 3.5 枚举值范围校验

不同字段档位必须使用各自白名单：

- 二档字段：`0/1/null`
- 三档字段：`0/1/2/null`
- 四档字段：`0/1/2/3/null`

枚举越界不能直接流入规则引擎。

### 3.6 null 值处理规则

`null` 表示字段 key 存在，但当前识别无法稳定给出枚举值。

- `null` 可以进入降级链。
- `null` 必须记录字段名、字段等级、来源和降级原因。
- 核心字段 `null` 数量过多时，应触发重试或重拍。
- 低置信字段 `null` 可优先使用默认降级值，但不能作为主母型核心支撑字段。

### 3.7 多余字段处理规则

多余字段不进入标准化层。

- VLM 原始输出中的 `image_quality` 与 `confidence` 可进入日志或质量分析，不进入 33 字段。
- 未定义字段必须丢弃并记录。
- 多余字段本身不必导致失败，除非它替代了必填字段或造成输出不可映射。

### 3.8 缺失字段处理规则

字段缺失不能直接流入规则引擎。

- 核心字段缺失优先触发 VLM 重试或几何处理重试。
- 辅助字段缺失可进入降级链，但必须记录。
- 低置信字段缺失可使用默认降级值，但不能影响主母型硬约束。

### 3.9 VLM 原始字段到内部 33 字段映射校验

映射层必须检查 VLM 原始字段是否能映射到对应内部大写字段。

- `main_lines` 映射 14 个字段。
- `special_patterns` 映射 2 个字段。
- `mounts` 映射 6 个字段。
- `overall` 映射 2 个字段。
- MediaPipe / 几何处理补齐 9 个非 VLM 字段。
- `FINGERTIP_SHAPE` 不来自 VLM Prompt，不能要求 VLM 输出该字段。

如果 VLM 输出无法映射到 33 字段，返回 `VLM_OUTPUT_UNMAPPABLE`。

### 3.10 降级字段数量统计

每次标准化必须统计：

- 总降级字段数。
- 连续降级字段数。
- 核心字段降级数。
- 辅助字段降级数。
- 低置信字段降级数。
- `null` 转默认值的字段数。
- 枚举异常导致重试或拒绝的字段数。

连续 5 个字段降级必须触发重拍分支。

## 4. 错误类型到处理方式

| 错误类型 | 典型情况 | 错误码 | 处理方式 | 是否可进入后续匹配 |
|---|---|---|---|---|
| JSON 不可解析 | 完全不是 JSON | `JSON_PARSE_FAILED` | 重试 VLM；仍失败则要求重新上传 | 否 |
| Schema 结构错误 | 顶层结构缺失或层级错乱 | `JSON_SCHEMA_INVALID` | 重试 VLM 或进入重拍分支 | 否 |
| 必填字段缺失 | 33 字段中缺 key | `MISSING_REQUIRED_FIELD` | 按字段等级重试或降级；核心字段大量缺失则重拍 | 否 |
| 字段名不在白名单 | 出现未定义大写字段或表外字段 | `JSON_SCHEMA_INVALID` | 丢弃多余字段并记录；若替代必填字段则失败 | 视必填字段是否完整 |
| 类型错误 | 字符串、布尔值、数组、对象 | `TYPE_MISMATCH` | 校验失败，重试 VLM 或重拍 | 否 |
| 枚举越界 | `4`、`-1`、`1.5` 等 | `ENUM_OUT_OF_RANGE` | 校验失败，重试 VLM 或重拍 | 否 |
| null 过多 | 总 `null` 达到阈值或核心字段 `null` 过多 | `TOO_MANY_NULL_FIELDS` | 触发重试或重拍 | 否 |
| 降级过多 | 连续 5 个字段降级 | `TOO_MANY_DEGRADED_FIELDS` | 要求重新上传更清晰图片 | 否 |
| 核心字段不足 | 核心字段无法支撑后续主母型硬约束 | `CORE_FIELDS_INSUFFICIENT` | 重试或重拍，不强行匹配 | 否 |
| VLM 输出不可映射 | 输出语义无法落到 33 字段 | `VLM_OUTPUT_UNMAPPABLE` | 重试 VLM；仍失败则重拍 | 否 |
| 低置信输出 | 置信度低但字段仍可降级 | `LOW_CONFIDENCE_OUTPUT` | 记录降级，可继续但不得伪装为高置信 | 视核心字段是否充足 |

## 5. 通过条件

字段对象只有同时满足以下条件，才可以进入后续匹配准备层：

- JSON 可解析。
- 33 个必填 key 全部存在。
- 字段名完全符合白名单。
- 所有值都是数字枚举或 `null`。
- 无枚举越界。
- 降级字段已记录。
- 连续降级字段少于 5 个。
- 核心字段充足。
- 低置信字段未被用作主母型核心支撑。
