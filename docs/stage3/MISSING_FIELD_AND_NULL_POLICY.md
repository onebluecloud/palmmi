# Palmmi Stage 3E 缺失字段与 null 处理策略

## 1. 目标

本文定义 Palmmi Stage 3E 对字段缺失、`null`、类型错误、枚举越界和 VLM 输出无法映射的处理策略。

3E 只定义策略，不写代码，不接 API，不做规则引擎，不做人格匹配。

## 2. 状态定义

| 状态 | 定义 | 严重程度 | 是否可直接进入后续匹配 |
|---|---|---|---|
| key 缺失 | 33 个必填字段中缺少字段名 | 高 | 否 |
| key 存在但 value = `null` | 字段存在，但无法稳定识别 | 中 | 否，需先进入降级链 |
| value 类型错误 | 值为字符串、布尔值、数组或对象 | 高 | 否 |
| value 超出枚举范围 | 值为负数、小数或超出档位 | 高 | 否 |
| VLM 输出无法映射到 33 字段 | 原始输出无法落到内部字段 | 高 | 否 |

## 3. 基本原则

- key 缺失比 `null` 更严重。
- `null` 可以进入降级链。
- 枚举越界不能直接使用。
- 类型错误必须视为校验失败。
- 缺失字段不能直接流入规则引擎。
- 核心字段大量缺失时，不应强行进入人格匹配。
- 低置信字段缺失时，可以使用默认降级值，但不能影响主母型硬约束。

## 4. key 缺失处理

key 缺失表示输出结构不完整。

处理规则：

| 字段等级 | 缺失处理 |
|---|---|
| 核心字段 | 优先重试 VLM 或几何处理；多项缺失时触发重拍；不得进入匹配 |
| 辅助字段 | 可重试；仍缺失时可按默认值降级，但必须记录 |
| 低置信字段 | 可按默认值降级，但必须记录，不得作为主母型核心支撑 |

内部标准化层最终必须保留 33 个 key。缺失 key 不能被静默忽略。

## 5. null 处理

`null` 表示字段 key 已存在，但当前识别没有稳定结果。

处理规则：

| 字段等级 | null 处理 |
|---|---|
| 核心字段 | 优先重试；少量可降级，降级后仍需标记；大量 `null` 触发重拍 |
| 辅助字段 | 可进入降级链，使用默认降级值并记录 |
| 低置信字段 | 可使用默认降级值并记录；不能作为主母型核心支撑 |

`null` 字段完成降级后，必须保留降级记录，不能伪装成高置信原始识别结果。

## 6. 类型错误处理

类型错误包括：

- 字符串数字，例如 `"2"`。
- 中文枚举，例如 `"明显"`。
- 布尔值，例如 `true`。
- 数组或对象。

处理规则：

- 类型错误必须视为校验失败。
- 不允许自动把字符串转数字。
- 不允许把布尔值转成 `0/1`。
- 类型错误字段不得直接进入降级链，必须先重试或重拍。

## 7. 枚举越界处理

枚举越界包括：

- 二档字段出现 `2` 或更大值。
- 三档字段出现 `3` 或更大值。
- 四档字段出现 `4` 或更大值。
- 任意字段出现负数。
- 任意字段出现小数。

处理规则：

- 枚举越界不能直接使用。
- 不允许截断、四舍五入或自动夹取到最近合法值。
- 应返回 `ENUM_OUT_OF_RANGE` 并进入重试或重拍分支。

## 8. VLM 输出无法映射处理

VLM 输出无法映射包括：

- 输出字段名与 VLM 原始 Schema 不匹配。
- VLM 输出人格、命运、解释文字，未输出字段 JSON。
- 输出结构缺少 `main_lines`、`special_patterns`、`mounts`、`overall` 中必要来源。
- 输出尝试包含 `FINGERTIP_SHAPE` 等不应由 VLM 输出的内部字段。

处理规则：

- 返回 `VLM_OUTPUT_UNMAPPABLE`。
- 优先重试 VLM。
- 连续失败后要求用户重新上传更清晰图片。

## 9. 字段等级策略

### 9.1 核心字段

核心字段共 21 个，是后续母型和人格判断的主支撑字段。

- 核心字段缺失或 `null` 应优先重试。
- 核心字段大量缺失时，不应强行进入人格匹配。
- 核心字段降级必须记录，后续判断不得把它当作高置信原始值。

### 9.2 辅助字段

辅助字段共 4 个，可以参与评分和人格细化，但不能单独决定主母型。

- 辅助字段缺失或 `null` 可进入降级链。
- 辅助字段降级后仍只能作为辅助信号。
- 辅助字段不得弥补核心字段不足。

### 9.3 低置信字段

低置信字段共 8 个，只做低权重加分、人格细化和文案润色。

- 低置信字段缺失或 `null` 可使用默认降级值。
- 低置信字段不能单独决定主母型。
- 低置信字段降级后不能作为主母型核心支撑字段。
- `FINGERTIP_SHAPE` 始终按低置信字段处理。

## 10. 建议阈值

3E 文档层先冻结以下阈值，后续如需调整必须走变更流程：

- 总 `null` 字段数达到 5 个：触发 `TOO_MANY_NULL_FIELDS`。
- 核心字段 `null` 或缺失达到 3 个：触发 `CORE_FIELDS_INSUFFICIENT`。
- 连续 5 个字段降级：触发 `TOO_MANY_DEGRADED_FIELDS` 并要求重拍。

以上阈值用于阻止低质量或不完整输出强行进入后续匹配。
