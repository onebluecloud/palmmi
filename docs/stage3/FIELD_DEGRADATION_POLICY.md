# Palmmi Stage 3E 字段降级策略

## 1. 目标

本文基于 V4.2 字段降级策略，定义 Palmmi Stage 3E 的字段降级链、默认中位数、记录要求和状态分类。

3E 只定义降级策略，不写业务代码，不接 API，不做规则引擎，不做人格匹配。

## 2. 降级链

字段处理按以下顺序执行：

1. 使用 VLM 原值或几何处理原值。
2. 置信度低时退化为更粗判断。
3. 仍不稳定时填默认中位数。
4. 连续 5 个字段降级触发重拍。

降级链不能跳过记录步骤。任何降级字段都必须留下字段名、原始值、降级值、字段等级、来源、原因和时间点。

## 3. 默认中位数

默认中位数按字段档位固定：

| 字段档位 | 允许值 | 默认中位数 |
|---|---|---:|
| 0/1 二档字段 | `0/1/null` | 1 |
| 0-2 三档字段 | `0/1/2/null` | 1 |
| 0-3 四档字段 | `0/1/2/3/null` | 2 |

默认中位数只能用于降级，不代表模型原始识别结果。

## 4. 降级触发条件

字段进入降级链的典型原因：

- 原始值为 `null`。
- VLM 分组置信度低。
- 图片清晰度不足但尚未达到拒绝条件。
- VLM 原始输出可映射，但字段不稳定。
- 低置信字段在当前图片中难以稳定判断。

类型错误和枚举越界不直接进入降级链，应先视为校验失败并触发重试或重拍。

## 5. 连续 5 个字段降级触发重拍

V4.2 中新增字段和低置信字段稳定性较差，尤其是掌丘、末端分叉和指尖形态相关字段。

因此 Palmmi Stage 3E 使用“连续 5 个字段降级触发重拍”的策略，避免单个低置信字段拖累体验，也避免大量降级字段伪装成有效识别。

触发后处理：

- 返回 `TOO_MANY_DEGRADED_FIELDS`。
- 状态标记为 `RETRY_REQUIRED`。
- 提示用户重新上传更清晰的手掌照片。
- 不进入后续匹配。

## 6. 降级记录要求

每个降级字段必须记录：

| 记录项 | 说明 |
|---|---|
| `field_name` | 33 字段大写字段名 |
| `source` | VLM 原始路径或 MediaPipe / 几何来源 |
| `confidence_tier` | 核心字段 / 辅助字段 / 低置信字段 |
| `original_value` | 原始值，可为 `null` |
| `degraded_value` | 降级后的默认中位数或粗判断值 |
| `degradation_reason` | 低置信、看不清、缺失、映射不稳定等 |
| `can_support_core_matching` | 是否允许作为主母型核心支撑 |

降级字段不能伪装成高置信原始识别结果。

## 7. 低置信字段限制

低置信字段包括：

- `HEAD_LINE_END_FORK`
- `FINGERTIP_SHAPE`
- `MOUNT_VENUS`
- `MOUNT_JUPITER`
- `MOUNT_SATURN`
- `MOUNT_APOLLO`
- `MOUNT_MERCURY`
- `MOUNT_LUNA`

低置信字段降级后：

- 不能作为主母型核心支撑字段。
- 不能单独决定主母型。
- 只能用于低权重加分、人格细化或文案润色。
- 不得覆盖核心字段不足的问题。

## 8. 状态分类

| 状态 | 定义 | 是否可继续后续匹配 |
|---|---|---|
| `VALID` | 所有字段合法，无需降级，核心字段充足 | 是 |
| `DEGRADED` | 存在少量降级字段，已记录，核心字段仍充足 | 是，但降级字段不得伪装成高置信 |
| `RETRY_REQUIRED` | 连续 5 个字段降级、核心字段不足或 null 过多 | 否 |
| `REJECTED` | JSON 无法解析、Schema 严重错误、输出不可映射或图片明显不可用 | 否 |

## 9. 与后续匹配的边界

降级策略只决定字段是否可进入后续匹配准备层，不执行规则引擎。

进入后续匹配前必须满足：

- 字段名仍为 33 个大写字段。
- 字段值为合法数字枚举。
- 降级字段已记录。
- 核心字段数量充足。
- 低置信字段未被用作主母型核心支撑。
